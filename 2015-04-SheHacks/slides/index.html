0. Install Meteor

Set up Meteor, Node and Mongo distributions. 

$ curl https://install.meteor.com/ | sh

To follow along, also clone the shehacksnyc repo 

$ git clone https://github.com/MeteorHudsonValley/shehacksnyc.git

Meteor is relatively stable on *nix systems. A preview version was just released for Windows *.
Once you clone the repo, you can grab the version for a given step by simply checking out the tag. The tag for each step will be clearly marked in the slide title. So a title with 'tags/step-1' implies you can switch to code for that step using 'git checkout tags/step-1' on your local cloned repo.

1. Scaffold an app tags/step-1

Meteor creates the basic boilerplate code for you. 

$ meteor create webapp

Open an editor. Let's explore the code. 

ls -al ./webapp
webapp.js     // app processing (contains data helpers, event handlers)
webapp.html   // app view (contains body, header, templates)
webapp.css    // app styles
.meteor       // meteor configuration, local db
Directory hierarchy, file names have meaning. The .meteor directory is managed by Meteor and should not be edited manually.

1a. Run The App

Meteor's isobuild toolchain creates a runnable app. 

$ cd webapp
$ meteor

=> Started proxy.                             
=> Started MongoDB.                           
=> Started your app.                          

=> App running at: http://localhost:3000/

Open a browser. Let's explore the app.

Boilerplate app shows a basic view (template), event handling (click) features. No styling (yet).
Use 'meteor run' explicitly to run against a specific platform e.g., 'meteor run android' runs app in an Android emulator.
Use 'meteor reset' to clear a local database and 'meteor mongo' to open a MongoDB console to it.

1b. Deploy The App

Meteor's isobuild toolchain also uploads the runnable app. 

$ meteor deploy shehacksnyc.meteor.com 
Deploying to shehacksnyc.meteor.com.          
Now serving at http://shehacksnyc.meteor.com 

$ meteor deploy shehacksnyc.meteor.com --delete
Deleted.

Meteor has free hosting for validation only, not for production apps. Deployment tied to your 'Meteor' account. Create one if needed.
For production, explore third party services. See Meteor Up (mup) for an example.

We have a basic hosted app!




Open http://shehacksnyc.meteor.com on mobile and desktop browsers. Leave those open..

2. Templates tags/step-2

are 'blueprints' (with data placeholders) for modular UI


Spacebars (language) for defining templates Blaze (engine) activates and renders template into DOM Tracker (library) triggers reactive DOM updates in Blaze on data changes


Templates are passive HTML structures with placeholders for data.
Template helpers bind data 'contexts' dynamically [ref], before it is rendered into DOM.
Template tags (#with, #each) can be used to set the data context explicitly.
Template events declare handlers for DOM events associated with that template.
